6# OVERALL DESCRIPTION
  
7>
  
8> ## Fantasy Baseball Statistics - "Play Ball!"
  
 9>
  
10> ### This is a Ruby CLI program for computing baseball statistics.
  
11>
  
12> The program runs and performs the following:
  
13>
  
14> 1.   Reads data into the program from input CSV files
  
15>
  
16> 2.   Computes statistics based on the Customer Specifications (see below)
  
17>
  
18> 3.   Outputs the aggregated statistics to the screen (STDOUT) for user to view
  
19>
  
20
  
21# USAGE
  
22>
  
23> ### To run:
  
24>
  
25     $ ./bin/play_ball
  
26>
  
27
  
28# INSTALLATION
  
29>
  
30> NOTE: In order to simplify the installation burden, these steps would normally be wrapped up in a rake task.  This was not done due to time constraints. Here are the manual instructions.
  
31>
  
32> Step 1: Clone repo
  
33>
  
34     $ git clone git@github.com:chrismcnabbsoftwaredeveloper/fantasy_baseball.git
  
35>
  
36
  
37> Step 2: Bundle
  
38>
  
39     $ bundle
  
40>
  
41
  
42> Step 3: Setup logging* - add this entry to <strong><em>/etc/syslog.conf</em></strong>
  
43>
  
44     local5.* /var/log/fantasy_baseball.log
  
45>
  
46
  
47> *Note: these steps are for Mac OSX. These steps will most certainly be different depending on your production environment.
  
48>
  
49> Step 3: Stop/Start syslogd daemon
  
50>
  
51     $ sudo launchctl unload /System/Library/LaunchDaemons/com.apple.syslogd.plist
  
52>
  
53     $ sudo launchctl load /System/Library/LaunchDaemons/com.apple.syslogd.plist
  
54>
  
55
  
56> Step 4: Change execute permissions on play_ball file
  
57>
  
58     $ chmod +x bin/play_ball
  
59>
  
60
  
61> Step 5: Setup/Verify input data files - configuration.rb
  
62>
  
63> By default, the csv input data files are located in the /data/ directory of the project.  However, this is configurable.  You will configure the file location in the following configuration file:
  
64>
  
65     $ ../config/configuration.rb
  
66>
  
67
  
68> For example,
  
69>
  
70     Configuration.for('batting_input_file') {
  
71       file_name 'data/Batting-07-12.csv'
  
72       }
  
73>
  
74
  
75> Step 6: Now to run the program,
  
76>
  
77     $ ./bin/play_ball
  
78>

80# CONFIGURATION
  
81>
  
82> ## Program Configuration Settings
  
83>
  
84> As mentioned above there are several configuration settings available inside the following file:
  
85>
  
86     $ ../config/configuration.rb
  
87
  
88>
  
89>
  
90> Here's the setting for the <strong><em>Player Roster CSV</em></strong> file:
91
  
91>
  
92     Configuration.for('player_roster') {
  
93       input_file_name 'data/Master-small.csv'
  
94     }
  
95
  
96>
  
97>
  
98> ...the setting for the <strong><em>Batting Data CSV</em></strong> file:
  
 99>
  
100
  
101     Configuration.for('batting_input_file') {
  
102       input_file_name 'data/Batting-07-12.csv'
  
103     }
  
104
  
105>
  
106>
  
107> ... the setting for the <strong><em>Pitching Data CSV</em></strong> file: (future)
  
108>
  
109     Configuration.for('pitching_input_file') {
  
110       input_file_name 'data/Pitching-07-12.csv'
  
111     }
  
112>
  
113>
  
114> ... the setting for the "csv_options" hash for CSV class used for data import (details: http://apidock.com/ruby/CSV):
  
115>
  
116
  
117     Configuration.for('csv_options') {
  
118       col_sep ','
  
119       row_sep :auto
  
120       quote_char '"'
  
121       field_size_limit nil
  
122       converters nil
  
123       unconverted_fields nil
  
124       headers true
  
125       return_headers false
  
126       write_headers false
  
127       skip_blanks false
  
128       force_quotes false
  
129       encoding 'UTF-8'
  
130     }

164# CUSTOMER SPECIFICATIONS
  
165>
  
166> Specification 1:
  
167> Display data for:
  
168>
  
169     "Most Improved Batting Average from 2009 to 2010"
  
170>
  
171
  
172> Specification 2:
  
173> Display data for:
  
174>
  
175     "Slugging Percentage for All Oakland A's Players in 2007"
  
176>
  
177
  
178> Specification 3:
  
179> Display data for:
  
180>
  
181     "The Triple Crown Winner for 2011 and 2012 for both the AL and the NL"
  
182>
  
183
  
184> Details regarding these customer specifications can be found in the spec doc located in the /doc/ folder of the project.
  
185>
  
186     $ ../doc/FantasyBaseballRequirements.doc
